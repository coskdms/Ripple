# 프로젝트 개요

## 한 줄 정의
> **"결정을 적고, 시간이 지나 다시 마주한다."**

## 앱의 본질
내가 내린 **결정을 기록**하고, 시간이 지난 후 **회고**하는 앱.
결정 하나가 물에 던진 돌처럼 삶 전체에 파문(Ripple)처럼 퍼져나간다.

## 이름
**Ripple** = 잔물결, 파문
- 결정 하나가 시간이 지나며 퍼져나가는 이미지
- 로고도 물결 원형으로 자연스럽게 연결

## 핵심 컨셉

### 결정 기록 항목 (6개)
| 항목 | 의미 | MVP 포함 |
|---|---|---|
| 상황 | 어떤 맥락에서 이 결정이 생겼나 | O |
| 선택지 | 어떤 옵션들이 있었나 (선택+포기) | v2 |
| 결정 | 무엇을 선택했나 | O |
| 이유 | 그때의 나는 왜 골랐나 | O |
| 감정 | 그때 느낌 (확신/두려움/설렘) | O |
| 기대 | 어떻게 될 거라 생각했나 | v2 |

### 회고 주기
회고 주기는 **고정값이 아니라 사용자가 켜고 끌 수 있는 설정값**이다.
기본 프리셋을 제공하되, 사용자가 원하는 단위로 회고할 수 있게 한다.

**기본 프리셋**
| 주기 | 성격 | 목적 |
|---|---|---|
| 일간 | 체크인 | 오늘 내린 결정들 빠르게 |
| 주간 | 체크인 | 결정 진행 상태 빠르게 체크 |
| 월간 | 회고/감성 | 결정 패턴 분석 ("이달엔 안정 3번, 도전 1번") |
| 연간/장기 | 회고/감성 | 감성적 하이라이트 ("1년 전 오늘의 결정") |

**주기별 성격 (기간만이 아니라 보여주는 것이 다르다)**
- 짧은 주기(일/주) → 체크인: 진행 상태, 빠르게 훑기
- 긴 주기(월/연) → 회고/감성: 패턴 분석, 그때 vs 지금

**사용자 커스텀 (v2)**
- 원하는 주기 직접 설정 (예: 3일마다, 2주마다, 6개월마다)
- 프리셋 개별 켜고 끄기
- 커스텀 주기는 길이에 따라 체크인/감성 성격 자동 적용

**알림 연계 (추후)**
- 회고 주기 설정은 리마인드 알림과 연결됨 (예: 일간 → 매일 저녁, 월간 → 매월 1일)
- MVP에서는 제외, 설계상 염두

### 회고를 특별하게 만드는 장치 (추후)
1. **결정 숙성도** — 미결 → 진행중 → 완숙 (시간 경과에 따라)
2. **포기한 선택지 추적** — 선택하지 않은 것이 어떻게 됐는지
3. **결과 평가** — 6개월/1년 후 "이 결정, 잘한 것 같아요?" (점수 아닌 감정)
4. **익명 공유** — 결정 구조만 익명으로 공유, 공감대 형성

## 목표 설정 구조 (추후 확장)

회고는 "기준"이 있어야 의미 있다. 단, **목표 설정이 진입 장벽이 되면 안 된다.**

```
Layer 1 — 나의 나침반 : 핵심 가치 3~5개, 되고 싶은 모습
Layer 2 — 목표 설정   : 장기 비전(3~5년) → 연간 → 분기
Layer 3 — 결정과 연결 : "이 결정은 어떤 목표/가치와 연결되나?"
```

### 사용자 유형별 온보딩
앱 첫 진입 시 3갈래로 분기:
- **A. 큰 목표가 있어요** → 목표 입력 → 결정 기록
- **B. 일단 결정부터 기록할래요** → 바로 기록 → 나중에 목표 연결
- **C. 제대로 설정하고 싶어요** → 가치관 → 비전 → 목표 → 결정

> 어떤 경로로 들어와도 결국 "결정 기록 → 회고"로 수렴한다.
