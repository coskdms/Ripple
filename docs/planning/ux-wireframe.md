# UX 와이어프레임 (MVP)

> 목적: **화면이 어떻게 구성되고 어떻게 이동하는지** 먼저 정해 개발 방향을 잡는다.
> 기준: **모바일 우선**(세로 화면). 데스크탑은 같은 구조를 넓게 펼치는 정도로 후순위.
> 범위: MVP 5기능(결정 기록·목록·상세·회고·인증)만. v2 요소는 자리만 표시(`(v2)`).

## 디자인 톤 (방향만)
- 컨셉: "잔물결(Ripple)" — 차분하고 여백 많은, 감성적인 톤. 화려함보다 **읽기 좋은 기록장** 느낌.
- 색: 차분한 단색 베이스 + 감정 표현용 포인트 컬러(감정별). 상세 색은 개발 단계에서.
- 폰트: 본문 가독성 우선. 회고(월/연)는 조금 더 큰 타이포로 "감성" 강조.

---

## 화면 목록 (MVP)

| # | 화면 | 경로(예상) | BFF | 비고 |
|---|---|---|---|---|
| S0 | 로그인 | `/login` | — | 구글 소셜 로그인만 |
| S1 | 홈 | `/` | `GET /api/home` | 오늘 결정 + 켜진 회고 묶음 |
| S2 | 결정 기록 | `/decisions/new` | `POST /api/decisions` | 4항목 폼 |
| S3 | 결정 목록 | `/decisions` | `GET /api/decisions` | 시간순, 페이징 |
| S4 | 결정 상세 | `/decisions/[id]` | `GET /api/decisions/{id}` | 삭제만 가능(수정 v2) |
| S5 | 회고 | `/retrospective` | `GET /api/retrospective` | 기간 선택 + flashback |
| S6 | 회고 설정 | `/settings/retrospective` | `GET/PUT /api/retrospective-settings` | 프리셋 on/off |

> 온보딩 3갈래(A/B/C 목표 설정 분기)는 **v2**. MVP는 로그인 후 바로 홈(=경로 B: 결정부터 기록).

---

## 화면 이동 흐름 (네비게이션)

```
            [S0 로그인]
                │ 로그인 성공
                ▼
        ┌─── [S1 홈] ───────────────┐
        │      │  │  │              │
   (기록 버튼) │  │  └─(회고 카드 탭)─┼──► [S5 회고]
        ▼      │  │                 │        │
   [S2 기록]   │  └─(설정 아이콘)────┼──► [S6 회고설정]
        │      │                    │
   저장 ▼      └─(목록 탭)──► [S3 목록] ──(항목 탭)──► [S4 상세]
   [S1 홈] 또는 [S4 상세]                                  │
                                                   삭제 ▼
                                                  [S3 목록]
```

- **하단 탭바(모바일 기본 네비)**: `홈` · `목록` · `(+) 기록` · `회고` · `설정`
  - 가운데 `(+)`를 강조(가장 핵심 행동 = 결정 기록).

## 화면별 레이아웃

### S0. 로그인 `/login`
```
┌─────────────────────────┐
│                         │
│        ◌ Ripple         │  ← 물결 로고 + 이름
│   결정을 적고,           │
│   시간이 지나 마주한다    │  ← 한 줄 슬로건
│                         │
│                         │
│  ┌───────────────────┐  │
│  │  G  Google로 시작  │  │  ← 구글 로그인 버튼(유일)
│  └───────────────────┘  │
│                         │
└─────────────────────────┘
```
- 요소: 로고, 슬로건, 구글 버튼 1개.
- 로그인 성공 → S1 홈. (브라우저—BFF 세션쿠키 발급)

---

### S1. 홈 `/`
```
┌─────────────────────────┐
│ ◌ Ripple        ⚙(설정) │  ← 헤더
│ 안녕하세요, OOO님         │
│─────────────────────────│
│  ✦ N개월 전 오늘         │  ← flashback 카드(있을 때만)
│  "당신은 이런 결정을…"    │     탭 → S5 회고
│─────────────────────────│
│ 오늘의 결정              │
│ ┌─────────────────────┐ │
│ │ [결정] 한 줄 요약     │ │  ← 오늘 기록한 결정 카드들
│ │ 😌 감정  · 오전 9:14 │ │     탭 → S4 상세
│ └─────────────────────┘ │
│ (없으면: "오늘 아직…")   │  ← 빈 상태 + 기록 유도
│─────────────────────────│
│ 켜둔 회고               │
│ [일간] [주간] [월간]…    │  ← 켜진 프리셋 칩, 탭 → S5
│─────────────────────────│
│ [홈][목록] (+) [회고][설정]│  ← 하단 탭바
└─────────────────────────┘
```
- 데이터: `GET /api/home` (유저정보 + 오늘 결정 + flashback + 켜진 회고). BFF가 묶어서 한 번에.
- 빈 상태: 오늘 결정 0개면 "오늘 내린 결정을 남겨볼까요?" + 기록 버튼.

### S2. 결정 기록 `/decisions/new`
```
┌─────────────────────────┐
│ ←        결정 기록        │  ← 뒤로 + 타이틀
│─────────────────────────│
│ 상황                     │
│ ┌─────────────────────┐ │  ← situation (멀티라인)
│ └─────────────────────┘ │
│ 결정                     │
│ ┌─────────────────────┐ │  ← choice
│ └─────────────────────┘ │
│ 이유                     │
│ ┌─────────────────────┐ │  ← reason (멀티라인)
│ └─────────────────────┘ │
│ 그때의 감정              │
│ (😌확신)(✨설렘)(🌊평온)  │  ← emotion: 8종 중 택1
│ (😮‍💨후련)(🌱희망)(🤔망설임)│
│ (😟불안)(😨두려움)        │
│ 한마디(선택)             │
│ ┌─────────────────────┐ │  ← emotion_note (nullable)
│ └─────────────────────┘ │
│ ┌───────────────────┐   │
│ │       저장        │   │  ← POST /api/decisions
│ └───────────────────┘   │
└─────────────────────────┘
```
- 항목: 상황·결정·이유(텍스트) + 감정(택1) + 감정 한마디(선택).
- 감정 선택지는 **고정 enum 8종**(확신·설렘·평온·후련함·희망·망설임·불안·두려움). 코드값은 `db-schema.md` 참고. v2: 선택지/기대 항목 추가 자리.
- 저장 성공 → **S4 상세(방금 기록한 결정)로 이동**. 적은 내용을 바로 확인시키는 흐름.
- **수정 없음**(MVP). 잘못 적으면 삭제 후 재작성.

---

### S3. 결정 목록 `/decisions`
```
┌─────────────────────────┐
│ 내 결정        [월][일]  │  ← 그룹 단위 토글(기본=월)
│─────────────────────────│
│ 2026년 6월               │  ← 그룹 헤더(시간순)
│ ┌─────────────────────┐ │
│ │ 6/6 [결정] 한 줄 요약 │ │  ← 카드: 날짜+요약+감정
│ │ 😌 확신             │ │     탭 → S4 상세
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 6/5 [결정] …        │ │
│ └─────────────────────┘ │
│        … (스크롤)        │  ← 페이징/무한스크롤
│─────────────────────────│
│ [홈][목록] (+) [회고][설정]│
└─────────────────────────┘
```
- 데이터: `GET /api/decisions` (시간순, 페이징).
- **그룹 단위 = 기본 월별**, 헤더 우측 토글로 **일별 보기 전환** 제공.
  - 월별: 헤더 "2026년 6월", 카드에 `6/6` 표기.
  - 일별: 헤더 "6월 6일(오늘)", 카드에 시각(`오전 9:14`) 표기.
  - 토글은 표시 방식만 바꿈(같은 데이터 재그룹핑). 서버 재호출 불필요.
- 빈 상태: "아직 기록한 결정이 없어요" + 기록 버튼.

### S4. 결정 상세 `/decisions/[id]`
```
┌─────────────────────────┐
│ ←                  ⋮     │  ← 뒤로 + 더보기(삭제)
│─────────────────────────│
│ 2026.06.06 오전 9:14     │  ← created_at
│ 😌 확신                  │  ← emotion (+ note 있으면 함께)
│─────────────────────────│
│ 상황                     │
│ …본문…                   │
│ 결정                     │
│ …본문…                   │
│ 이유                     │
│ …본문…                   │
│─────────────────────────│
│ (v2) N개월 후 결과 평가   │  ← 자리만, 비활성
│─────────────────────────│
│   ⋮ → [ 삭제 ]           │  ← 삭제 확인 모달
└─────────────────────────┘
```
- 데이터: `GET /api/decisions/{id}`.
- 액션: **삭제만**(`DELETE`). 삭제 → 확인 모달 → S3 목록.
- 수정 버튼 없음(MVP). v2 자리: 결과 평가/숙성도.

---

### S5. 회고 `/retrospective`
```
┌─────────────────────────┐
│ 회고                ⚙    │  ← 설정(S6) 진입
│ [일간][주간][월간][연간]  │  ← 기간 탭(켜진 프리셋만)
│─────────────────────────│
│ ◀  2026년 6월  ▶         │  ← 기간 네비게이션
│─────────────────────────│
│ ✦ 1년 전 오늘            │  ← flashback(감성), 긴 주기일수록 강조
│ "그때 당신은…"           │
│─────────────────────────│
│ 이 기간의 결정 (N개)     │
│ ┌─────────────────────┐ │
│ │ [결정] 요약 · 😌     │ │  ← 기간 내 결정 카드 → S4
│ └─────────────────────┘ │
│ (월/연) 감정 분포 요약    │  ← "안정 3 · 도전 1" 같은 패턴
│─────────────────────────│
│ [홈][목록] (+) [회고][설정]│
└─────────────────────────┘
```
- 데이터: `GET /api/retrospective` (기간 조회 + flashback, BFF가 묶음).
- **flashback 기준**: 오늘 날짜 기준 과거 같은 날의 결정. **"N년 전 오늘" 우선**,
  없으면 **"N개월 전 오늘"로 폴백**. (초반엔 개월 단위로 채워주고, 데이터가 쌓이면 년 단위 우선.)
- 주기 성격:
  - 일/주(체크인) → 결정 목록 위주, 빠르게 훑기.
  - 월/연(감성) → flashback·감정 분포 강조, 타이포 크게.
- (v2) AI 패턴 분석 코멘트 자리.

---

### S6. 회고 설정 `/settings/retrospective`
```
┌─────────────────────────┐
│ ←      회고 설정          │
│─────────────────────────│
│ 어떤 주기로 회고할까요?    │
│ 일간   체크인         [○]│  ← 토글 on/off
│ 주간   체크인         [○]│
│ 월간   회고/감성       [●]│
│ 연간   회고/감성       [●]│
│─────────────────────────│
│ (v2) + 커스텀 주기 추가   │  ← 자리만, 비활성
│ (v2) 알림 받기           │
└─────────────────────────┘
```
- 데이터: `GET/PUT /api/retrospective-settings` (프리셋 on/off).
- 토글 변경 즉시 저장(PUT). 켜진 프리셋이 S1 홈·S5 회고 탭에 반영.

## 열린 질문 (개발 전 확정 필요)
- ✅ 감정 enum 8종 확정 (확신·설렘·평온·후련함·희망·망설임·불안·두려움). 코드값은 `db-schema.md`.
- ✅ 저장 후 이동 위치: **상세(S4)** 로 확정.
- ✅ 목록 그룹 단위: **기본 월별 + 일별 토글 전환** 제공.
- ✅ flashback 기준: **개월 포함 + 년 우선**("N년 전 오늘" 우선, 없으면 "N개월 전 오늘" 폴백).

> 열린 질문 4건 모두 확정. 와이어프레임 1차 완료 상태.
```
